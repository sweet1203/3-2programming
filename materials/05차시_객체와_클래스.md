## 🌟 도입 - 붕어빵으로 이해하는 클래스와 객체

여러분, 붕어빵 좋아하시나요? 🐟🥟

붕어빵을 만들 때 어떻게 만들죠?

1. 먼저 **붕어빵 틀(설계도)** 이 있어요. → 이것이 **클래스(Class)**!
2. 그 틀로 **여러 개의 붕어빵**을 찍어내요. → 이것이 **객체(Object)**!

```
   붕어빵 틀 (클래스)         붕어빵들 (객체)
   ┌─────────────┐            🐟  🐟  🐟
   │   〰️〰️〰️    │   →       팥  슈크림 피자
   │  🐟 모양    │           
   └─────────────┘
```

> 💡 **클래스 = 설계도**, **객체 = 그 설계도로 만든 실제 물건**

## 📚 핵심 개념 1: 객체란?

객체는 **속성**과 **메소드**를 함께 가진 단위예요.

### 🔹 우리 주변에서 객체 찾기 🔍

**예시: 스마트폰**
- **속성(어떤 상태인지):** 모델명, 색상, 배터리 잔량, 저장 공간
- **메소드(어떤 행동을 할 수 있는지):** 전화걸기(), 사진찍기(), 메시지보내기()

**예시: 우리 학교 학생**
- **속성:** 이름, 학번, 학년, 키, 좋아하는 과목
- **메소드:** 공부하기(), 출석하기(), 시험치기()

> 💡 **속성**은 **명사**(데이터), **메소드**는 **동사**(행동)예요!

## 📚 핵심 개념 2: 클래스 만들기

### 🔹 기본 구조

```python
class 클래스이름:
    def __init__(self, 매개변수들):    # 생성자 (객체 만들 때 자동 실행)
        self.속성1 = 값1
        self.속성2 = 값2
    
    def 메소드이름(self):
        실행할 코드
```

> 📌 **꼭 기억!**
> - 클래스 이름은 보통 **대문자**로 시작 (예: `Student`, `Car`)
> - `__init__`은 **언더스코어 2개** 양쪽에 있어요!
> - `self`는 **자기 자신을 가리키는** 단어 (다른 언어의 `this`와 비슷)

### 🔹 간단한 예시

```python
class Dog:
    def __init__(self, name, age):    # 강아지를 만들 때 이름과 나이를 받음
        self.name = name              # 이름 속성
        self.age = age                # 나이 속성
    
    def bark(self):                   # 짖는 메소드
        print(f'{self.name}: 멍멍!')

# 객체 만들기 (강아지 두 마리!)
dog1 = Dog('초코', 3)
dog2 = Dog('두부', 5)

# 속성 확인
print(dog1.name)    # 초코
print(dog2.age)     # 5

# 메소드 호출
dog1.bark()         # 초코: 멍멍!
dog2.bark()         # 두부: 멍멍!
```

### 🔍 코드 풀이
- `Dog('초코', 3)` → `Dog` 클래스로 객체를 만들면서 이름과 나이를 전달
- `__init__`이 자동으로 실행되면서 `self.name = '초코'`, `self.age = 3` 설정됨
- `dog1.bark()` → `dog1` 객체의 `bark` 메소드 호출

## 💻 핵심 코드: 학생 클래스 만들기

```python
class Student:
    def __init__(self, name, grade):
        self.name = name      # 이름 속성
        self.grade = grade    # 학년 속성
    
    def introduce(self):
        print(f'안녕하세요! 저는 {self.grade}학년 {self.name}입니다.')
    
    def study(self, subject):
        print(f'{self.name}이(가) {subject} 공부 중!')

# 객체 만들기 (Student 클래스로 학생 2명 만들기)
s1 = Student('민지', 2)
s2 = Student('서연', 1)

# 속성 확인
print(s1.name)    # 민지
print(s2.grade)   # 1

# 메소드 호출
s1.introduce()    # 안녕하세요! 저는 2학년 민지입니다.
s2.study('수학')   # 서연이(가) 수학 공부 중!
```

### 🔍 코드 흐름 따라가기
1. `Student('민지', 2)` 실행 → `__init__`이 자동으로 실행됨
2. `self.name = '민지'`, `self.grade = 2` 가 저장됨
3. `s1.introduce()` 호출 → `self`가 자동으로 `s1`을 가리킴
4. 출력 완료!

### 사례 3: 은행 계좌 클래스 🏦
```python
class Account:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance
    
    def deposit(self, amount):
        self.balance += amount
        print(f'{amount}원 입금. 잔액: {self.balance}원')
    
    def withdraw(self, amount):
        if amount <= self.balance:
            self.balance -= amount
            print(f'{amount}원 출금. 잔액: {self.balance}원')
        else:
            print('잔액 부족!')

# 계좌 만들기
my_account = Account('민지', 10000)
my_account.deposit(5000)    # 5000원 입금. 잔액: 15000원
my_account.withdraw(3000)   # 3000원 출금. 잔액: 12000원
my_account.withdraw(20000)  # 잔액 부족!
```

```
   ├─ __init__ : 객체 생성 시 자동 실행
   ├─ self : 자기 자신을 가리킴
   └─ . (점) : 속성/메소드 접근

5. 속성 vs 메소드
   ├─ 속성(attribute): 데이터 (예: 이름, 나이)
   └─ 메소드(method): 행동 (예: 짖기, 달리기)
```
