## 🌟 도입 - 붕어빵으로 이해하는 클래스와 객체

여러분, 붕어빵 좋아하시나요? 🐟🥟

붕어빵을 만들 때 어떻게 만들죠?

1. 먼저 **붕어빵 틀(설계도)** 이 있어요. → 이것이 **클래스(Class)**!
2. 그 틀로 **여러 개의 붕어빵**을 찍어내요. → 이것이 **객체(Object)**!

```
   붕어빵 틀 (클래스)         붕어빵들 (객체)
   ┌─────────────┐            🐟  🐟  🐟
   │   〰️〰️〰️    │   →       팥  슈크림 피자
   │  🐟 모양    │           
   └─────────────┘
```

> 💡 **클래스 = 설계도**, **객체 = 그 설계도로 만든 실제 물건**

## 📚 핵심 개념 1: 객체란?

객체는 **속성**과 **메소드**를 함께 가진 단위예요.

### 🔹 우리 주변에서 객체 찾기 🔍

**예시: 스마트폰**
- **속성(어떤 상태인지):** 모델명, 색상, 배터리 잔량, 저장 공간
- **메소드(어떤 행동을 할 수 있는지):** 전화걸기(), 사진찍기(), 메시지보내기()

**예시: 우리 학교 학생**
- **속성:** 이름, 학번, 학년, 키, 좋아하는 과목
- **메소드:** 공부하기(), 출석하기(), 시험치기()

> 💡 **속성**은 **명사**(데이터), **메소드**는 **동사**(행동)예요!

## 📚 핵심 개념 2: 클래스 만들기

### 🔹 기본 구조

```python
class 클래스이름:
    def __init__(self, 매개변수들):    # 생성자 (객체 만들 때 자동 실행)
        self.속성1 = 값1
        self.속성2 = 값2
    
    def 메소드이름(self):
        실행할 코드
```

> 📌 **꼭 기억!**
> - 클래스 이름은 보통 **대문자**로 시작 (예: `Student`, `Car`)
> - `__init__`은 **언더스코어 2개** 양쪽에 있어요!
> - `self`는 **자기 자신을 가리키는** 단어 (다른 언어의 `this`와 비슷)

### 🔹 간단한 예시

```python
class Dog:
    def __init__(self, name, age):    # 강아지를 만들 때 이름과 나이를 받음
        self.name = name              # 이름 속성
        self.age = age                # 나이 속성
    
    def bark(self):                   # 짖는 메소드
        print(f'{self.name}: 멍멍!')

# 객체 만들기 (강아지 두 마리!)
dog1 = Dog('초코', 3)
dog2 = Dog('두부', 5)

# 속성 확인
print(dog1.name)    # 초코
print(dog2.age)     # 5

# 메소드 호출
dog1.bark()         # 초코: 멍멍!
dog2.bark()         # 두부: 멍멍!
```

### 🔍 코드 풀이
- `Dog('초코', 3)` → `Dog` 클래스로 객체를 만들면서 이름과 나이를 전달
- `__init__`이 자동으로 실행되면서 `self.name = '초코'`, `self.age = 3` 설정됨
- `dog1.bark()` → `dog1` 객체의 `bark` 메소드 호출

## 💻 핵심 코드: 학생 클래스 만들기

```python
class Student:
    def __init__(self, id, name):
        self.id = id                    # 학번 속성
        self.name = name                # 이름 속성
        self.rented_books = []          # 빌린 책 리스트 (처음엔 비어 있음)
    
    def rent_book(self, book):
        """책을 빌리는 메소드"""
        if not book.is_rented:           # 책이 대여 중이 아니면
            book.is_rented = True
            self.rented_books.append(book)
            return f'{self.name}에게 {book.title} 대여함.'
        else:
            return f'{book.title}은 이미 대여 중입니다.'

class Book:
    def __init__(self, title):
        self.title = title
        self.is_rented = False           # 처음엔 대여 안 됨

# 객체 생성
s1 = Student(1, '홍길동')
b1 = Book('파이썬 완전정복')
b2 = Book('데이터 분석 입문')

# 책 빌리기
print(s1.rent_book(b1))    # 홍길동에게 파이썬 완전정복 대여함.
print(s1.rent_book(b1))    # 파이썬 완전정복은 이미 대여 중입니다.
print(s1.rent_book(b2))    # 홍길동에게 데이터 분석 입문 대여함.
```

class Car:

### 사례 3: 은행 계좌 클래스 🏦
```python
class Account:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance
    
    def deposit(self, amount):
        self.balance += amount
        print(f'{amount}원 입금. 잔액: {self.balance}원')
    
    def withdraw(self, amount):
        if amount <= self.balance:
            self.balance -= amount
            print(f'{amount}원 출금. 잔액: {self.balance}원')
        else:
            print('잔액 부족!')

# 계좌 만들기
my_account = Account('민지', 10000)
my_account.deposit(5000)    # 5000원 입금. 잔액: 15000원
my_account.withdraw(3000)   # 3000원 출금. 잔액: 12000원
my_account.withdraw(20000)  # 잔액 부족!
```

- ① 객체를 삭제할 때 실행되는 메소드
- ② 객체가 생성될 때 자동으로 실행되는 메소드 (생성자) 
- ③ 객체의 정보를 출력하는 메소드
- ④ 클래스를 정의하는 메소드

<details><summary>정답 보기</summary>

**정답: ②**  
`__init__`은 객체가 만들어질 때 자동으로 실행되어 초기 속성을 설정해요.
</details>

### Q4. 다음 코드의 출력 결과는?

```python
class Cat:
    def __init__(self, name):
        self.name = name
    def meow(self):
        print(f'{self.name}: 야옹!')

c = Cat('나비')
c.meow()
```
- ① `name: 야옹!`
- ② `나비: 야옹!` 
- ③ `Cat: 야옹!`
- ④ 오류

**정답: ② 나비: 야옹!**  
`Cat('나비')`로 객체를 만들 때 `self.name = '나비'`가 설정되고, `meow()`에서 출력돼요.
</details>

### Q5. 다음 중 설명이 옳은 것은?
- ① 속성은 행동, 메소드는 데이터다
- ② 속성은 데이터, 메소드는 행동이다

- ④ 객체에는 속성만 있고 메소드는 없다
